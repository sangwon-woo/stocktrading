Surfing
현재 만들고 있는 주식 알고리즘 트레이딩 시스템
로컬(LG Gram notebook)에서 개발한 후 Long Beach(server)로 이관할 예정
Long Beach는 주식 트레이딩용 서버
이후 코인 트레이딩용 서버는 GCP 또는 AWS를 사용할 예정

객체
1. 투자자(User) : 나, 성주, 다른 사람들 등 투자금을 Surfing에 투입하는 주체
2. Surfing : 투자자로부터 자본을 받아 주식 알고리즘 트레이딩을 수행하는 주체
  2-1. 투자자 테이블 : 투자자 정보가 담겨 있는 테이블
    2-1-1. user_id
    2-1-2. user_name
    2-1-3. user_account_number
    2-1-4. 등록 날짜
    2-1-5. 투자 날짜
    2-1-6. 초기투자금액
  2-2. 투자자 내역 테이블
3. 투자현황
4. 거래내역
5. 전략현황

원천 테이블
1. 일별 데이터
2. 분봉 데이터
3. 종목 데이터
4. 전략 데이터

구현되어야 하는 기능

1. 증권사에서 데이터 송수신
2. 데이터베이스에서 데이터 송수신
3. 위험관리 기능
4. 자산관리 기능
5. 투자평가 기능(Performance)
